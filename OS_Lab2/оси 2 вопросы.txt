позикс - портативный интерфейс ос - стандарты вз-ия между ос и прикладной программой
позикс-совемстимый код может быт ьскомпилен под любую позикс-совместимую систему

прерывание - запрос на остановку текущего вып. кода, чтобы выполнить определенное
действие, описанное в обработчике прерывания

системколл - обращение к функциям ядра от прикладной програмимы
методом программного прерывания (старый - int, новый - systemcall[amd] systementer [intel])

процесс - единица работы ОС, создается по системколлу, соотв. прлграмме (ехе), иозлированы
друг от друга, имеет PID и PPID, механизм медпроцессного вз-ия для вз-ия между ними (IPC).
IPC безальтернативен (че бля?)
процесс = объект ядра + адресное пр-во. при его создании ему выделяется адресное пр-во.
программу создает загрузчик. получить процессы - lsof, ps, top или tasklist и Taskkill
при остановке процесса вся инфа созраняется в контексте, чтобы продолжить. это надо 
для прерывания. 
сисиетмные процессы находятся в рамках ядра стартует с ос (демоны\службы) без вз-ия с юзером
systemctl - вз-ие с демонами

возрастанеи еадресов code, static, heap, data, stack
вектор прерываний - массив на 0-ом адресе

системколл для создания проца в винде - CReateProcess
в линуксе вроде fork()
